**Claims Quest** is a Duolingo-style browser game designed to train users on the intricacies of the healthcare claims adjudication process. ## üïπÔ∏è Game Levels

### Unit 1: The Intake Gate (Form Finder)
- **Objective**: Sort incoming claim forms into the correct processing bins.
- **Mechanic**: Drag and Drop.
- **Logic**: 
  - **CMS-1500** ‚Üí Professional Claims
  - **UB-04** ‚Üí Facility/Admission Claims

### Unit 2: The Submission Scrubber
- **Objective**: Validate claims against strict submission rules.
- **Mechanic**: Speed Round (True/False).
- **Rules**:
  - Hand-written claims are **ALWAYS** rejected.
  - Claims with white-out/alterations are **REJECTED**.
  - Clean digital claims with valid NPI are **ACCEPTED**.

### Unit 3: Informatics Engine
- **Objective**: Identify and tag key data elements on a claim form.
- **Mechanic**: Point & Click / Tagging.
- **Key Elements**: Member Info, Provider Info, COB, Procedure Codes, Diagnosis Codes.
- **Bonuses**:
  - Find the **Taxonomy Code** to earn the **Specialist Badge**.
  - Identify **Modifiers** to unlock payment logic insights.

### Unit 4: The Adjudication Gauntlet
- **Objective**: Correctly sequence the rigorous 7-step adjudication workflow.
- **Mechanic**: Sortable List Puzzle.
- **Critical Logic**: "Pricing Edits" (reducing billed charges) **MUST** occur before "Complete Adjudication".
